## 系统概览

### 研究背景与问题

现有水下光学通信系统普遍采用**固定参数设计**——发射功率、调制参数、接收增益在通信建立后保持不变。这一设计在实验室清水环境下运行良好，但在真实水下环境中会面临：

- **水体浑浊度动态变化**（浮游生物、泥沙再悬浮）
- **光源与相机之间距离变化**（水下设备漂移）
- **背景光干扰波动**（水面折射光、人工光源）

固定参数系统无法应对上述动态因素，导致**通信中断或误码率急剧上升**。

### 核心问题

> 如何在浑浊动态水下环境中实现具备"感知-反馈"闭环的自适应光学通信？

### 解决方案

本系统构建三级自适应机制：

```
┌─────────────────────────────────────────────────────────────┐
│                    自适应控制闭环                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   感知层 ──► 算法层 ──► 执行层 ──► 反馈 ──► 感知层          │
│   (相机)     (自适应    (LED/    (信道   (重新               │
│             控制算法)   增益)    质量)   感知)                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 OOK 调制解调

#### 调制端（发射侧）

```
原始比特流:  1 0 1 1 0 0 1 1

                ▼

```

#### 解调端（接收侧）

```
相机采集 ──► 帧提取 ──► 阈值判决 ──► 比特恢复
                              ▲
                              │
                        阈值自适应
                        (依据信道状态调整)
```

#### IEEE 802.15.7 符合性

本系统遵循 IEEE 802.15.7 光学通信标准的关键参数：

| 参数 | 标准值 | 本系统实现 |
|------|-------|----------|
| 调制方式 | OOK / PPM | OOK ✅ |
| 符号速率 | 1-100 kbaud | 可配置 ✅ |
| RLL 编码 | 4B6B / 8B10B | 4B6B ✅ |
| 前向纠错 | Reed-Solomon | RS(255,239) 🔄 |

### 2.2 RLL 编码

采用 **4B6B** 线路编码，将 4 位数据扩展为 6 位传输：

```
原始数据:  [b3 b2 b1 b0]
              │
              ▼
        4B6B 编码表
              │
              ▼
        传输符号:  [t5 t4 t3 t2 t1 t0]
```

**编码规则（部分）**

| 原始 4-bit | 编码 6-bit | 直流平衡 |
|-----------|-----------|---------|
| 0000 | 001111 | +2 |
| 0001 | 010011 | 0 |
| 0010 | 011001 | 0 |
| ... | ... | ... |
| 1111 | 110000 | -2 |

直流平衡的目的：长时间传输后线路电平均值为零，便于接收端 AC 耦合。

## 研究主线三：自适应控制算法

> **这是本系统的核心创新点**

### 3.1 ISO 增益优化（Matus 2020 复现）

**参考论文**：Matus et al., "Adaptive Gain Control for Underwater Optical Communication", 2020

**核心思想**：接收信号强度（RSSI）反馈调节发射功率或接收端 ISO 增益。

**算法流程**：

```
┌─────────────────────────────────────────────┐
│           ISO 增益自适应控制                 │
├─────────────────────────────────────────────┤
│                                             │
│  设定目标 RSSI_target                        │
│         │                                   │
│         ▼                                   │
│  测量当前 RSSI_current                       │
│         │                                   │
│         ▼                                   │
│  计算误差 e = RSSI_target - RSSI_current     │
│         │                                   │
│         ▼                                   │
│  ΔGain = Kp × e                            │
│         │                                   │
│         ▼                                   │
│  ISO_new = ISO_old + ΔGain                  │
│         │                                   │
│         ▼                                   │
│  等待下一帧，循环                             │
│                                             │
└─────────────────────────────────────────────┘
```

**仿真结果**（已完成 ✅）：

- 收敛时间：约 15 帧
- 稳态误差：< 5%
- 适用场景：轻度浑浊、慢变信道

### 3.2 阻尼多状态算法（曝光+增益联合控制）

**来源**：马的工作（实验室积累算法）

**核心思想**：单一 ISO 增益无法覆盖从清水到重度浑浊的宽动态范围，引入**曝光时间**作为第二控制变量，形成多状态机。

**状态机定义**：

```
                    ┌─────────┐
                    │ STABLE  │◄────────────────┐
                    │ 稳定态  │                 │
                    └────┬────┘                 │
                         │ 收到扰动              │
            ┌────────────┼────────────┐         │
            │            │            │         │
            ▼            ▼            ▼         │
      ┌──────────┐ ┌──────────┐ ┌──────────┐   │
      │  LOW     │ │  HIGH    │ │ DARK     │   │
      │ 低增益态  │ │ 高增益态  │ │ 低光态    │   │
      └────┬─────┘ └────┬─────┘ └────┬─────┘   │
           │            │            │         │
           └────────────┼────────────┘         │
                        │  阻尼切换（防振荡）    │
                        └──────────────────────┘
```

**状态切换条件**：

| 当前状态 | 条件 | 切换目标 |
|---------|------|---------|
| STABLE | `|error| > threshold_high` | LOW / HIGH |
| STABLE | `|error| > threshold_dark` | DARK |
| LOW/HIGH/DARK | `|error| < threshold_low` | STABLE |

**阻尼机制**：

```
状态切换不是瞬间完成，而是带阻尼系数 α：

ISO_new = α × ISO_target + (1-α) × ISO_old

α 越小，切换越慢，越不容易振荡。
典型值：α = 0.3
```

**仿真框架**：已在 `mimo/` 目录下搭建 ✅，正在调参 🔄

### 3.3 MIMO 合并（双目相机）

**背景**：单接收器 MIMO 系统利用**空间分集**对抗多径衰落。

**三种合并策略**：

#### EGC（等增益合并）

```
r_EGC = Σ w_i × r_i  （|w_i| = 1, 相位对齐）
```

#### SC（选择性合并）

```
选择 SNR 最高的分支作为输出：

r_SC = r_k  其中 k = argmax(SNR_i)
```

#### MRC（最大比合并）

```
最优合并，各分支按 SNR 加权：

r_MRC = Σ (SNR_i / Σ SNR_j) × r_i
```

**仿真框架**：

```
mimo/
├── egc.py          # 等增益合并
├── sc.py           # 选择性合并
├── mrc.py          # 最大比合并
├── channel.py      # 水下信道模型（待完善）
└── simulation.py   # 仿真主程序
```

**仿真参数**（当前设置）：

| 参数 | 值 |
|------|---|
| 信道模型 | Rayleigh 衰落 |
| 基带速率 | 10 kbps |

| 水深模拟 | 3 m |

## 常见问题与解决

### Q: 浑浊度增加时 BER 急剧上升怎么办？

**A**: 两层应对：
1. 短期：提高发射功率（切到 10W 支路）+ 拉高 ISO
2. 中期：切换到 MIMO MRC 合并，充分利用空间分集
3. 长期：引入前向纠错（Reed-Solomon）对抗随机错误

### Q: 双目相机同步问题？

**A**: 硬件同步方案：用硬件 trigger 同步两路相机曝光。
软件同步方案：如果同步误差 < 1 符号周期，可以在 MIMO 合并时做时域对齐。

### Q: 10W LED 散热怎么处理？

**A**: 铝基板 + 导热硅脂 + 小风扇。实测连续工作 30 分钟，温升约 20°C，在可接受范围内。
